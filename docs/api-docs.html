Documentação da API

Esta documentação descreve os endpoints disponíveis na API do Sistema de Quantificação Price Action Brooks. A API permite acessar e gerenciar dados de padrões, trades, estatísticas e mais.

A URL base para todos os endpoints é: https://api.priceactionsystem.com/v1

Requisitos

Para utilizar a API, você precisará:

  • Um token de acesso válido (veja a seção de Autenticação)
  • Enviar solicitações usando HTTPS
  • Incluir os cabeçalhos necessários com cada solicitação

Autenticação

Todas as solicitações à API devem incluir um token de autenticação JWT no cabeçalho. O token pode ser obtido através do endpoint de login.

POST /auth/login

Obtenha um token de autenticação.

Corpo da Requisição

Parâmetro Tipo Descrição
email string Endereço de e-mail registrado
password string Senha da conta

Exemplo de Requisição

POST /auth/login HTTP/1.1
Host: api.priceactionsystem.com
Content-Type: application/json

{
  "email": "usuario@exemplo.com",
  "password": "senha123"
}

Exemplo de Resposta

HTTP/1.1 200 OK
Content-Type: application/json

{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 3600,
    "user": {
      "id": "123456",
      "name": "Usuário de Exemplo",
      "email": "usuario@exemplo.com"
    }
  }
}

Usando o Token

Inclua o token em todas as solicitações subsequentes no cabeçalho Authorization: 

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Tratamento de Erros

A API utiliza códigos de status HTTP padrão para indicar o sucesso ou falha de uma requisição. Em caso de erro, o corpo da resposta contém informações detalhadas sobre o problema.

Códigos de Status

Código Descrição
200 - OK A requisição foi bem-sucedida
201 - Created Um novo recurso foi criado com sucesso
400 - Bad Request A requisição é inválida ou malformada
401 - Unauthorized Autenticação necessária ou inválida
403 - Forbidden Sem permissão para acessar o recurso
404 - Not Found Recurso não encontrado
422 - Unprocessable Entity Validação falhou
429 - Too Many Requests Limite de requisições excedido
500 - Internal Server Error Erro interno do servidor

Formato de Erro

Em caso de erro, a API retorna uma resposta no seguinte formato: 

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Dados inválidos",
    "details": [
      {
        "field": "email",
        "message": "E-mail inválido"
      }
    ]
  }
}

Objeto Padrão

Estrutura do objeto Padrão retornado pela API.

Campo Tipo Descrição
ID string Identificador único do padrão
Nome string Nome do padrão
Categoria string Categoria do padrão (ex: Reversão, Continuação)
Descrição string Descrição detalhada do padrão
Força string Classificação de força (A, B ou C)
Probabilidade Base string Probabilidade base de sucesso
Contexto Ideal string Contexto ideal para usar este padrão
created_at string Data de criação no formato ISO
updated_at string Data da última atualização no formato ISO

Objeto Trade

Estrutura do objeto Trade retornado pela API.

Campo Tipo Descrição
ID string Identificador único do trade
Data e Hora string Data e hora do trade no formato ISO
Padrão Brooks string Nome do padrão utilizado
Par de Moedas string Par de moedas ou ativo negociado
Timeframe string Timeframe do trade
Contexto de Mercado string Contexto de mercado durante o trade
Direção string Direção do trade (Long ou Short)
Qualidade do Setup string Avaliação da qualidade do setup (A, B ou C)
Entrada string Preço de entrada
Stop string Preço de stop loss
Alvo string Preço alvo
Resultado (R) string Resultado em múltiplos de R
Observações string Observações sobre o trade
created_at string Data de criação no formato ISO
updated_at string Data da última atualização no formato ISO
